# Project Scaffolding Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Initialize a Nuxt 4 SPA project with a Cloudflare Pages Function worker, shared TypeScript types, and linting config — zero business logic, just the skeleton.

**Architecture:** Nuxt 4 SPA builds to `dist/` for Cloudflare Pages static hosting. A Pages Function at `functions/api/parse-confluence-page.ts` handles the API endpoint. Shared types live in `shared/types.ts` — importable from both Nuxt frontend and the Pages Function without bundling issues. Linting uses ESLint flat config with Nuxt's preset.

**Tech Stack:** Nuxt 4 (SPA mode), pnpm, Cloudflare Pages + Functions, TypeScript, ESLint flat config, Prettier

### Task 4: Create shared TypeScript types

**Files:**
- Create: `shared/types.ts`

- [ ] **Step 1: Create shared/types.ts**

```typescript
// ---------------------------------------------------------------------------
// AgendaFlow — shared types used by both the Nuxt frontend and the
// Cloudflare Pages Function.
// ---------------------------------------------------------------------------

// A single topic with its timebox.
export interface AgendaTopic {
  title: string
  /** Duration in minutes. Default 15 when the AI can't determine one. */
  durationMinutes: number
  /** Optional extra context extracted from the page. */
  notes?: string
}

// The parsed meeting agenda returned by the AI.
export interface MeetingAgenda {
  title: string
  topics: AgendaTopic[]
}

// ---------------------------------------------------------------------------
// POST /api/parse-confluence-page
// ---------------------------------------------------------------------------

export interface ParseConfluencePageRequest {
  url: string
}

export type ParseConfluencePageResponse =
  | { ok: true; agenda: MeetingAgenda }
  | { ok: false; code: ParseErrorCode; message: string }

export type ParseErrorCode =
  | 'INVALID_URL'
  | 'NOT_CONFLUENCE_CLOUD'
  | 'CONFLUENCE_FETCH_FAILED'
  | 'DEEPSEEK_API_ERROR'
  | 'PARSE_FAILED'
  | 'UNEXPECTED_ERROR'
```

- [ ] **Step 2: Add path alias in tsconfig.json so both frontend and functions can import from `#shared/*`**

Open `tsconfig.json`. Add (or extend) `compilerOptions.paths`:

```json
{
  "compilerOptions": {
    "paths": {
      "#shared/*": ["./shared/*"]
    }
  }
}
```

If `tsconfig.json` already has a `paths` object, merge the new entry into it. If `tsconfig.json` extends another config (e.g. `./.nuxt/tsconfig.json`), add a top-level `compilerOptions` block with `paths` — TypeScript merges them.
